üìã Database Schema (users table)

  • userID (PRIMARY KEY, AUTO_INCREMENT)
  • Email (VARCHAR, UNIQUE, NOT NULL)
  • Password (VARCHAR, NOT NULL) - Hashed with password_hash()
  • FirstName, LastName, FullName, Phone (VARCHAR, NULL)
  • Role (ENUM: 'Admin', 'User', DEFAULT 'User')
  • isVerified (BOOLEAN, DEFAULT false)
  • profileCompleted (BOOLEAN, DEFAULT false)
  • verificationToken (VARCHAR(64), NULL)
  • tokenExpires_at (DATETIME, NULL)
  • created_at, updated_at (TIMESTAMP)

üîë Session Variables

  • $_SESSION['userId'] - User's unique ID
  • $_SESSION['email'] - User's email address
  • $_SESSION['firstName'], $_SESSION['lastName'], $_SESSION['fullName']
  • $_SESSION['role'] - 'Admin' or 'User'
  • $_SESSION['isVerified'] - Email verification status (true/false)

‚úÖ Key System Rules

  • Users must verify email before logging in
  • Unverified users get a NEW verification email on login attempt
  • Verification tokens expire after 1 hour
  • Profile completion is required on first login after verification
  • All protected pages check: session validity + verification + profile completion
  • Passwords are hashed with password_hash() using PASSWORD_DEFAULT
  • Session must be destroyed on logout
